# Task 12 – Windows Security Log Analysis (Part A)

## Overview
This task demonstrates how to analyze Windows Security Event Logs to
identify failed login attempts using Event Viewer. The task was performed
for cybersecurity learning and awareness in a controlled environment.

Only **Part A** of the task was completed. **Part B was intentionally
excluded** as per instructions.

---

## Tools and Environment
- Operating System: Windows 11
- Tool Used: Windows Event Viewer

---

## Objective
- Understand Windows Security Event Logs
- Identify failed login attempts
- Use Event ID filtering for log analysis
- Gain hands-on experience with Event Viewer

---

## Methodology
1. Opened Event Viewer from Windows search
2. Navigated to **Windows Logs → Security**
3. Used **Filter Current Log**
4. Applied **Event ID 4625**
5. Reviewed filtered security events
6. Captured screenshots for documentation

---

## Files Included
- `analysis.txt` – Technical observations and findings
- `report.md` – Detailed security log analysis report
- `screenshots/` – Evidence of steps performed

---

## Outcome
This task provided practical experience in Windows security monitoring.
It demonstrated how failed authentication attempts can be detected using
native Windows tools without third-party software.
